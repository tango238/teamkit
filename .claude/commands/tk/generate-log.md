
# セットアップ

1.  **`baseDir` の設定**: `specs`
2.  **`specDir` の取得**: スラッシュコマンドの第1引数を読み取ります。
    -   引数がない場合は、エラーメッセージ「エラー: `specDir` 引数が必要です。使用法: `/tk-update-status <specDir> <commandName> <versionNumber>`」を表示し、**実行を停止**します。
3.  **`commandName` の取得**: スラッシュコマンドの第2引数を読み取ります。
4.  **`versionNumber` の取得**: スラッシュコマンドの第3引数を読み取ります。

# 実行

`baseDir`、`specDir`、`commandName`、`versionNumber` を使用して、以下の手順を実行してください。

**重要**:
- ユーザーへの出力（ステータスメッセージ、完了通知）はすべて**日本語**で行ってください。
- ユーザーに確認を求めずにファイルを保存してください。

# 実行ログ作成コマンド

## 目的

コマンドの実行ログを `{{baseDir}}/{{specDir}}/logs/{{versionNumber}}/{{commandName}}.md` に保存します。
ユーザーの確認を求めずに、以下のプロセスを即座に実行してください。

## 実行手順

### 1. ログディレクトリとファイルの作成
- ログディレクトリを確認します: `{{baseDir}}/{{specDir}}/logs/{{versionNumber}}`
- ディレクトリが存在しない場合は作成します。
- ログファイルを作成します: `{{baseDir}}/{{specDir}}/logs/{{versionNumber}}/{{commandName}}.md`
    - ファイルの内容には、ヘッダーとして実行日時とコマンド名を記述してください。

### 2. 前回の実行詳細ログの取得
- このプロンプトの呼び出し元の実行詳細ログの取得してログ情報として、1で作成したログファイルに記載してください。

### 3. 結果の表示
以下の形式で結果を表示してください:

```
ステータス: 成功
コマンド: {{commandName}}
バージョン: {{versionNumber}}
日時: <現在のタイムスタンプ (ISO 8601)>
ログファイル: {{baseDir}}/{{specDir}}/logs/{{versionNumber}}/{{commandName}}.md
メッセージ: ログファイルを作成しました。
```
